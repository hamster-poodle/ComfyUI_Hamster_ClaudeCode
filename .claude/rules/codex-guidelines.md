# Codex セカンドオピニオン ガイドライン

`openai/codex-plugin-cc` をセカンドオピニオンとして活用するルール。

## いつ使うか

### 推奨タイミング
- アプローチが 2 回以上失敗したとき
- 複数の選択肢で迷ったとき
- 実装後にコードレビューを受けたいとき
- 専門外の領域で自信がないとき

## コマンド一覧

| コマンド | 用途 |
|----------|------|
| `/codex:review` | コードレビューを依頼 |
| `/codex:adversarial-review` | 設計判断に対してより批判的なレビュー |
| `/codex:rescue` | バグ調査・修正タスクを委託 |
| `/codex:status` | 実行中ジョブの進捗確認 |
| `/codex:result` | 完了ジョブの結果取得 |
| `/codex:cancel` | 実行中ジョブのキャンセル |

## 注意

- Codex の回答は参考意見。最終判断は自分で行う。
- 機密情報（APIキー・パスワード等）を含むコードをそのまま渡さない。
- bash 環境では `codex` コマンドが PATH に含まれないため、内部実行に失敗する場合がある。
  回避策: `export PATH="$PATH:/c/Users/paras/AppData/Roaming/npm"` を `~/.bashrc` に追加する。
